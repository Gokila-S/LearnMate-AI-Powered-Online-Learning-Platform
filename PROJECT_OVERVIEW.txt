LearnMate — Project Overview
Repository: https://github.com/Gokila-S/LearnMate-AI-Powered-Online-Learning-Platform/

1) Executive Summary
---------------------
LearnMate is an AI‑augmented online learning platform implemented on the MERN stack (MongoDB, Express, React, Node). The project aims to provide a modular learning experience: course catalog, module/lesson structure, multiple lesson types (video, YouTube, text, quizzes, assessments), enrollment and progress tracking, plus AI capabilities to assist students (chat widget) and speed content authoring (AI-generated quiz/assessment suggestions). The platform supports local media uploads and optional Google Drive uploads (service account or OAuth personal account). Authentication is JWT-based with role separation for website admins, course admins and learners.

2) Value proposition / Audience
--------------------------------
- Audience: Developers, instructors, content authors, and organizations wanting a self-hosted learning platform with AI tooling.
- Value: Rapid content authoring with AI assistance, flexible media handling (local & Drive), role-based access and an admin toolkit for management and moderation.

3) Top-level Architecture
-------------------------
- Frontend: Vite + React application (client-side routing via react-router-dom). Tailwind CSS for styling. Components include course listing, course detail, lesson player, admin editors, AI chat widget.
- Backend: Node.js + Express REST API (server.js / app.js). Uses Mongoose for MongoDB models. File uploads handled by multer; optional Google Drive integration via googleapis.
- Database: MongoDB (local or Atlas) accessed through `MONGODB_URI`.
- Authentication: JWT tokens signed with `JWT_SECRET` and validated in middleware.
- Integrations: OpenAI for chat and quiz generation; Google Drive (service account or OAuth) for uploads; Razorpay for payments (optional features present).

6) Features (detailed)
-----------------------
- Course browsing: search, category filters, level filters, alphabetical sorting
- Enrollment system: users can enroll in courses; progress is tracked
- Lessons: multiple types
  - Video (local upload with multer, optional upload to Google Drive)
  - YouTube lessons (store & play via ID)
  - Text lessons (Markdown/HTML rendered on client)
  - Quizzes & assessments: authorable, with quiz player UI and assessment flows
- Lesson progress: watched seconds saved; completion thresholds (60% early, 90% auto-complete)
- Bookmarks, badges, and completion tracking
- AI features:
  - Student Chat Widget (OpenAI) — contextual Q&A
  - Admin AI quiz/assessment generation — difficulty control, regenerate, "more like this", dedupe (Jaccard overlap filtering)
- Media pipeline: local uploads, optional Drive uploads (Service Account / OAuth personal), Drive fallback to local if issues
- Admin tools: course/module/lesson management, provider applications, moderation tools
- Payments: integration points for Razorpay (payment flows not necessarily fully wired in UI)

7) Data Model Highlights
------------------------
- `Course` -> contains `Module[]`
- `Module` -> contains `Lesson[]`
- `Lesson` -> `content` object with `type` and `data`; for video includes `storage` flag and `driveFileId` when stored in Drive
- Progress store: per-user watch entries with fields `watchedSeconds`, `durationSeconds`, timestamps

12) Common issues & troubleshooting
----------------------------------
- MongoDB connection errors: verify `MONGODB_URI`, ensure service running or Atlas network allowlist.
- CORS / API access: ensure `CLIENT_URL` matches frontend and backend CORS settings.
- OpenAI errors: ensure `OPENAI_API_KEY` is set and not rate-limited.
- Drive upload errors: ensure service account has permissions or run OAuth flow; verify private key newline format.
